Use other JavaScript libraries

When a third-party library is used during development, add the library in the init.js file, which is under the stemapp folder.

If a library is shared by many widgets, place the library under the root folder of the app, and add the library link into the resources array.

resources = resources.concat([
  {type: 'css', url: apiUrl + 'js/dojo/dojo/resources/dojo.css'},
  {type: 'css', url: apiUrl + 'js/dojo/dijit/themes/claro/claro.css'},
  {type: 'css', url: apiUrl + 'js/esri/css/esri.css'},
  {type: 'js', url: apiUrl},
  {type: 'css', url: path + 'jimu.js/css/jimu.css'}
]);

If you want to load the library in a widget only, use yepnope to load it. For more information, see http://yepnopejs.com/.

If the library is an AMD package, you can configure the package at dojoConfig.init.js.

dojoConfig = {
    parseOnLoad: false,
    async: true,
    tlmSiblingOfDojo: false,

    has: {
      'extend-esri': 1
    },
    packages : [{
      name : "widgets",
      location : path + "widgets"
    }, {
      name : "jimu",
      location : path + "jimu.js"
    }, {
      name : "themes",
      location : path + "themes"
    }]
  }
