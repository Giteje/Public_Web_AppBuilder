Frequently Asked Questions

How to launch Web AppBuilder for ArcGIS

On Windows, double click the startup.bat file inside the unzipped folder. Web AppBuilder starts in your default browser. Web AppBuilder runs on the top of Node.js with default port of 3344. It is not necessary to install Node.js separately as it is contained in the ZIP file.

On other platforms, open the command prompt window, navigate to the server directory, then type: node server.js. Once the Node.js starts, launch Web AppBuilder in the browser with http://[yourmachinename]:3344/webappbuilder. 3344 is the default port for Node.js.

If you want to run Web AppBuilder with additional parameters or different port , follow the instructions below:

  1. Open a command window and navigate to the server directory of Web AppBuilder installation.
    • On 64bit Windows, type: node_x64.exe server.js [-port=value] [-proxy=value]
    • On 32bit Windows, type: node_x32.exe server.js [-port=value] [-proxy=value]
    • On other platforms, type: server.js [-port=value] [-proxy=value]
    Note:

    -port specifies the port you want Node.js to run on, and -proxy specifies the proxy URL if needed.

  2. Launch Web AppBuilder in the browser with http://[yourmachinename]:[port]/webappbuilder, where [port] is the port number specified in step 1.

How to launch Web AppBuilder in https website with your CA-signed certificate

Web AppBuilder uses a self-signed certificate in Nodejs to support https. if you want to launch Web AppBuilder in your full https site, you may use your own certificate. To do so, two files in server directory should be replaced: cakey.pem and cacert.pem. Refer to http://nodejs.org/api/https.html and https://www.openssl.org/docs/HOWTO/certificates.txt for more information.

How to use the help documentation

You can launch the Web AppBuilder help in a browser by http://[yourmachinename]:[port]/webappbuilder/help after Node.js starts. Or you can go to the docs folder of Web AppBuilder installation, right click index.html file and open it with FireFox or IE. If you choose the latter option, be aware that Chrome does not support this html format while with IE10+, you need to enable JavaScript before continuing.

What should I do when I fail to log in my ArcGIS Online organization or Portal for ArcGIS

You may fail to log in to your ArcGIS Online organization or Portal for ArcGIS due to one of the followings:

  • No trusted certification available. To work around it, log in your portal first to make the certificate trusted, then log in Web AppBuilder again.
  • Need proxy to access externally. To work around it, set the proxy for Web AppBuilder from node command node server.js [-proxy=value].

Is SSL enabled access supported?

Yes. Web AppBuilder supports SSL enabled access to Online organization or Portal for ArcGIS. The https request is required when specify the URL to such organization or Portal for ArcGIS.

Which portal securities are supported

Portal with following securities are supported by Web AppBuilder:

  • OAuth
  • SAML
  • IWA
  • PKI
  • LDAP

Is portal with disconnected environment supported

Yes. You must set your own apiURL manually in env.js file. Refer to Change URL of ArcGIS API for JavaScript. By default, Web AppBuilder points to the ArcGIS API for JavaScript hosted by ArcGIS.com, such as http://js.arcgis.com/3.11.

Deployed app can't access cross domain resources

You need to set appropriate proxy in app config.json file. Refer to Use proxy.

Do I have to register my downloaded app for deployment

The app created by Web AppBuilder can be downloaded and hosted on your web server. Any app that is downloaded must be registered with ArcGIS Online or Porta for ArcGIS if the app references non-public items. Refer to Deploy app.

Which versions of the Portal for ArcGIS should I use?

Web AppBuilder supports Portal for ArcGIS 10.3 and up. If you have Portal for ArcGIS 10.2.2 or 10.2.1, it is recommended to upgrading it to 10.3.

Can I use the Portal for ArcGIS 10.2.2 and 10.2.1?

Although Web AppBuilder should work with Portal for ArcGIS 10.2.1 and 10.2.2, you are on your own risk if you prefer to use them as they are not supported.

Which versions of ArcGIS API for JavScript should I use

Web AppBuilder supports ArcGIS API for JavaScript 3.11 and up. The 3.11 version is available by ArcGIS Online. It also comes with the Portal for ArcGIS 10.3.

Which versions of ArcGIS Server should I use?

As Portal for ArcGIS 10.3 supports older versions of ArcGIS Server, Web AppBuilder should work with ArcGIS Server 10.0 and up.
