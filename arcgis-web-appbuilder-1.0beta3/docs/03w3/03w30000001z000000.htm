Chart

The Chart widget displays one or more quantitative attributes for a feature as a graphical representation of data. It is designed to make it easy for end users to observe possible patterns and trends in quantitative attribute data, as charts can usually be read more quickly than the raw data from which they are produced.

The following table shows the attributes and descriptions.

Attribute

Description

charts

Object[]. There is no default. -Array of chart task. Each element has the following most child attributes, but the specific attributes depend on chart mode. Look at config example for details.

  • name: String; There is no default— The name of chart tasks displayed in the widget.
  • url: String; There is no default. -A feature layer URL for the layer you want to perform chart task.
  • filter: Object. There is no default. -The 'expr' attribute of filter takes the form of a basic SQL statement and uses it to filter features.
  • description: String. There is no default. -The description of chart task.
  • mode: String. There is no default. -The mode of chart task. The available values are 'feature','category','count' and 'field'. If 'feature', chart task displays values feature by feature; If 'category', chart task displays values by category; If 'count', chart task displays feature counts by category; If 'field', chart task displays attribute values as charts.
  • types: String[]. There is no default -The types of chart task. The available values are 'column','pie','bar' and 'line'.
  • symbol: Object. There is no default -This is the feature symbol of chart task.
  • highLightColor: String. There is no default -The highlight color of selected features.
  • colors: String[]. There is no default. -Colors used to render charts.
  • labelField: String; There is no default— The field used as chart axis label. This attribute exists when mode is 'feature'.
  • sortOrder: String. default: 'asc' -How to sort category field value or label field value. The available values are 'asc' and 'des'. 'asc' means ascending order and 'des' means descending order. This attribute exists when mode is 'feature','category' or 'count'.
  • valueFields: String[]. There is no default -The fields provide values to statistics. This attribute exists when mode is 'feature','category' or 'field'.
  • categoryField: String. There is no default -The field that is used to categorize the chart. This attribute exists when mode is 'category' or 'count'.
  • operation: String. There is no default. -The attribute indicates which math method the chart task uses to calculate statistics. The available values are 'average','sum','max' and 'min'. This attribute exists when mode is 'category' or 'field'.
